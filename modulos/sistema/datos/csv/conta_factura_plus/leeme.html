Migración de datos de FacturaPlus y ContaPlus

  1. Pasos previos
    • Realizar las copias de seguridad pertinentes
    • Localizar el directorio de FacturaPlus donde están los ficheros DBF con los datos
    • Localizar el directorio de ContaPlus donde están los ficheros DBF con los datos
    • Copiar todos los DBF de FacturaPlus a un directorio nuevo
    • Copiar al mismo directorio los ficheros de ContaPlus diario.dbf y subcta.dbf
  2. Crear una base de datos nueva
  3. Cargar los módulos
    • sistema - datos
    • facturación - principal.
      Generar la empresa por defecto y el ejercicio.
      Ajustar las fechas y dígitos de subcuenta de este ejercicio a las correspondientes a los datos a importar
    • facturación - almacén. En general NO tendremos que abrir el módulo de Almacén para crear el almacén por defecto.
    • facturación - facturación
    • facturación - tesorería
    • contabilidad - principal. NO generar el plan general contable
  4. Desde el módulo sistema/datos, importar el módulo de datos con el directorio de los esquemas
  5. Desde el módulo sistema/datos, abrir el formulario maestro de ficheros y establecer la ruta de datos al directorio donde están los DBF.
  6. Crear las series de facturación correspondientes a los datos a importar
  7. Ejecutar el proceso especial Generar Ficheros CSV. En un determinado momento puede parecer que se para. Darle su tiempo
  8. Ejecutar en el directorio de los CSV este script para formatear los caracteres
  9. Ejecutar el proceso especial Fichero de Tipos de Rappel
  10. Ejecutar el proceso especial Completar Bancos
  11. Seguir el siguiente ORDEN en la ejecución de procesos:
    • Paises (si no existe el fichero BANDERA.dbf introducir los paises a mano)
    • Bancos
    • Departamentos
    • Divisas
    • Impuestos
    • Tipos de Rappel
    • Familias Artículos
    • Agentes Comerciales
    • Grupos de clientes
    • Intervalos de rappel
    • Sucursales bancarias. Al importar las sucursales bancarias es posible que algunos bancos no estén incluidos en la tabla de bancos por defecto
    • Sucursales Bancarias de Clientes
    • Sucursales Bancarias de Proveedores
    • Almacenes
    • Subcuentas
  12. Generar el PCG con el mismo número de dígitos de las subcuentas previamente importadas
  13. Ejecutar el proceso especial Actualizar Subcuentas
  14. Ir al módulo principal de financiera. Se ejecutará el proceso de registro de las subcuentas de cliente y proveedor
  15. Ejecutar el proceso especial Fichero de Asientos
  16. Seguir el siguiente ORDEN en la ejecución de procesos:
    • Asientos
    • Cuentas bancarias
    • Formas de Pago
    • Articulos
    • Stocks
    • Remesas de recibos
    • Partidas
  17. Ejecutar el proceso especial Actualizar Saldos
  18. Seguir el siguiente ORDEN en la ejecución de procesos:
    • Clientes
    • Cuentas bancarias de clientes
    • Direcciones de clientes
    • Proveedores
    • Albaranes de Clientes
    • Albaranes de Proveedores
    • Facturas de Clientes
    • Facturas de Proveedores
    • Lineas de Albaranes de Clientes
    • Lineas de Albaranes de Proveedores
    • Lineas de Facturas de Clientes
    • Lineas de Facturas de Proveedores
    • Pedidos de Clientes
    • Pedidos de Proveedores
    • Presupuestos de Clientes
    • Recibos de Clientes
    • Cuentas bancarias de proveedores
    • Direcciones de proveedores
    • Lineas de Pedidos de Clientes
    • Lineas de Pedidos de Proveedores
    • Lineas de Presupuestos de Clientes
    • Pagos y Devoluciones de Recibos de Clientes
  19. Ejecutar los siguientes procesos especiales:
    • Completar Facturas y Recibos
    • Valores de Facturación
    • Numeros de Facturacion
    • Plazos de Pago
    • Comentarios y Observaciones
  20. Cargar los módulos restantes (informes, etc) y comenzar a usar FacturaLUX con normalidad

Migración de datos sólo de ContaPlus

  1. Pasos previos
    • Realizar las copias de seguridad pertinentes
    • Localizar el directorio de ContaPlus donde están los ficheros DBF con los datos. Los ficheros necesarios son diario.dbf y subcta.dbf
  2. Crear una base de datos nueva
  3. Cargar los módulos
    • sistema - datos
    • facturación - principal.
      Generar la empresa por defecto y el ejercicio.
      Ajustar las fechas y dígitos de subcuenta de este ejercicio a las correspondientes a los datos a importar
    • contabilidad - principal. NO generar el plan general contable
  4. Desde el módulo sistema/datos, importar el módulo de datos con el directorio de los esquemas
  5. Desde el módulo sistema/datos, abrir el formulario maestro de ficheros y establecer la ruta de datos al directorio donde están los DBF.
  6. Ejecutar el proceso especial Generar Ficheros CSV. En un determinado momento puede parecer que se para. Darle su tiempo
  7. Ejecutar en el directorio de los CSV este script para formatear los caracteres
  8. Ejecutar el proceso Subcuentas
  9. Generar el PCG con el mismo número de dígitos de las subcuentas previamente importadas
  10. Ejecutar el proceso especial Actualizar Subcuentas
  11. Ejecutar el proceso especial Fichero de Asientos
  12. Seguir el siguiente ORDEN en la ejecución de procesos:
    • Asientos
    • Partidas
  13. Ejecutar el proceso especial Actualizar Saldos
  14. Ejecutar el proceso especial Fichero de Clientes y Proveedores / Subcuentas. Es interesante abrir los ficheros generados en este proceso con un editor de texto y eliminar los registros correspondientes a subcuentas que no son de clientes/proveedores (430000000, 40000000, etc)
  15. Seguir el siguiente ORDEN en la ejecución de procesos:
    • Clientes desde subcuentas
    • Direcciones de clientes desde subcuentas
    • Proveedores desde subcuentas
    • Direcciones de proveedores desde subcuentas
    Estos procesos importan clientes correspondientes a las subcuentas 43... y proveedores correspondientes a las subcuentas 40... y 41...

    los códigos de cliente y proveedor corresponderán a la numeración de las subcuentas.

  16. Ir al módulo principal de financiera. Se ejecutará el proceso de registro de las subcuentas de cliente y proveedor
  17. Cargar los módulos restantes (informes, etc) y comenzar a usar FacturaLUX con normalidad
Antes de comenzar...

· La estructura de datos de ContaPlus y FacturaPlus es distinta a la de FacturaLUX, por lo que no es posible volcar directamente los ficheros de datos de aquellos programas en las tablas de FacturaLUX. Es necesario alternar la importación directa mediante Procesos con la ejecución de Procesos Especiales. Estos ajustan los datos al sistema FacturaLUX, importan datos no "importables" con los procesos y crean algunos ficheros necesarios.

· Es importante realizar todos los pasos en el orden indicado para evitar problemas de incoherencia en las relaciones de tablas. Si en algún punto se necesita vaciar las tablas, puede ser necesario vaciar previamente las tablas relacionadas siguiendo el orden inverso al de la importación.

· Los datos de origen se encuentran en ficheros DBF. FacturaLUX convierte los BDF en CSV. Si es necesario algún cambio en los datos para resolver problemas al importar se pueden tocar los CSV con un editor de texto. Trabajaremos con un directorio en el que estarán todos los ficheros DBF originales y los nuevos CSV creados por FacturaLUX.

· Para evitar problemas después de la importación, el campo "ignorar contabilidad" se pone a verdadero en todas las facturas y pagos/devoluciones importados. La contabilidad correspondiente a la facturación importada debe existir previamente en ContaPlus, y será también importada. Después de su importación, la modificación o eliminación de estos datos NO repercutirá en la contabilidad.

· Los datos se importan sólo para un ejercicio, el ejercicio marcado como actual en el formulario de Empresa. Para importar un nuevo ejercicio deberá crearse en FacturaLUX, establecerse como actual y repetir el proceso completo (excepto obviamente la creación de la nueva base de datos)

· Es inevitable la pérdida de alguna información en el proceso global ya que sencillamente algunos campos de ContaPlus y FacturaPlus no existen en FacturaLUX (y viceversa).

· Si sólo se necesita importar la facturación o la contabilidad, puede seguirse este proceso realizando sólo los pasos correspondientes

· El proceso es largo y tedioso, y es posible que se interrumpa por incoherencias en los datos a importar. La importación de las partidas contables puede tardar horas cuando se trata de miles de partidas

Script para corregir caracteres

clear;
echo;
for i in *.csv
do

echo Sustitutendo en $i

cat $i | sed "s/ß/á/g" > tmp.tmp
cat tmp.tmp > $i

cat $i | sed "s/Ó/é/g" > tmp.tmp
cat tmp.tmp > $i
done

rm tmp.tmp -f
